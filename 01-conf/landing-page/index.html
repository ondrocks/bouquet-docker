If you can read this page, it means that Bouquet is working properly.

What have you just installed? A lot:

  • Bouquet server components, hosted as webapps in a Tomcat server
  • Apache2 as the web frontend
  • MongoDB, used to store metadata about your analytics
  • Redis, which is used as a cache and makes things run faster
Bouquet also brings an internal Elasticsearch server, that will automatically index your facets and provide you the best filtering experience using your data.

I just authenticated, but now I see openbouquet.io on my address bar , not my Bouquet installation!

Great, now you're authenticated. But don't worry, you're still using your Bouquet instance. See that ?apiUrl= at the end of the address ? That's it. That's the address of the Docker container that hosts your own Bouquet server.

That's the power of Bouquet's API. You can use whatever javascript application from whatever website you want. If this application uses Bouquet's javascript SDK, you can use the apiUrl parameter to point to the Bouquet server you need.

The javascript application does not directly access the data. That's why you don't even need to get it via HTTPS. Your browser downloads the application, and the locally-executed javascript connects to the Bouquet server hosted in your Docker container.
